Single-Cycle RISC-V Processor

Project Overview

The objective of this project was to design and implement a functional 64-bit single-cycle RISC-V processor using the Digital simulation environment. This architecture executes every instruction within a single clock cycle, ensuring all data movements and computations are completed before the next clock tick.

The processor was built to handle a variety of computational, control flow, and memory-access instructions, transitioning from a basic manual circuit into an autonomous system capable of executing complex programs automatically.

System Architecture

The top-level architecture is organized around a Program Counter (PC) that holds the address of the instruction currently being executed. On every clock cycle, the PC typically increments by 4 bytes to fetch the next 32-bit machine code instruction from ROM.

Component Breakdown

The execution core features a 64-bit Arithmetic Logic Unit (ALU) capable of addition, subtraction, multiplication, and logical shifts. Specialized logic using splitters and MUXes facilitates sub-word loading and storing, allowing the system to handle 8-bit (lb/sb) and 32-bit (lw/sw) values within the 64-bit memory framework.

Instruction Decoder: Utilizes a spreadsheet-based methodology to map instruction opcodes and function codes to specific control lines such as RFW (Register File Write) and ALUOp.

Instruction Set Support

  • R-type: Arithmetic operations like add, sub, mul.
  • I-type: Immediate operations like addi and ld.
  • Control Flow: jal, jalr, and conditional branches (beq, bne, blt, bge).
  • Memory: ld/sd, lw/sw, and lb/sb.

Development Methodology

The project followed an incremental strategy: Initial focus on the basic data path (PC, ROM, RegFile), followed by the automated Instruction Decoder, and finally the complex load/store logic for 64-bit RAM interfacing.

Debugging and Verification

Verification was conducted using a dedicated "dashboard" that exposes the internal state of all 32 registers, mapping them to their ABI names (a0, ra, sp). This allowed for single-stepping through programs to validate control lines in real-time.
